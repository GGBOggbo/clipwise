# 直播回放智能切片工具｜前端 UI 设计稿 v0.2

## 1. 产品定位

这是一个面向知识类主播的 AI 直播回放切片工具。

用户选择本地 MP4 直播回放后，系统基于音频转写和 AI 分析，推荐高价值切片。用户可以预览候选片段、编辑标题/摘要/原文金句/字幕，并快速导出 MP4 + SRT + 文案；需要时也可以生成带字幕视频。

这个产品不是专业剪辑软件，也不是营销官网。

核心目标是：

> 让知识主播不用完整看完 1-2 小时直播，也能快速找到并导出值得二次发布的短视频素材。

---

## 2. 设计原则

1. 工具型优先，不做复杂剪辑软件。
2. 页面要清楚、可信、低压力。
3. 不要暴露太多技术细节。
4. 不要让用户误以为原始视频上传到了服务器。
5. 默认帮用户判断，但保留人工确认。
6. 用户主路径必须顺：选回放 → 分析 → 看候选 → 预览 → 编辑 → 导出。

---

## 3. 页面结构

MVP 只做两个核心页面：


暂不做：

```text
工作台

素材库

复杂剪辑页
```

# 7. 候选片段定义

## 7.1 什么是候选片段

候选片段不是最终成片。

候选片段指：

> AI 基于直播转写文本识别出的、可能值得二次发布的时间段。

它的作用是帮用户从 1-2 小时直播中优先圈出值得看的部分。

## 7.2 候选片段判断标准

一个候选片段通常需要满足：

```text
1. 长度大约 60-180 秒
2. 观点、案例、方法或结论相对完整
3. 用户不看前后文也能基本理解
4. 信息密度较高
5. 废话比例较低
6. 有标题、金句或传播点
7. 适合被二次发布
```

## 7.3 候选片段类型

每个候选片段显示一个主类型。

类型包括：

```text
观点
案例
避坑
方法
金句
对比
结论
```

示例：

```text
[案例] 推荐
[观点] 强推荐
[方法] 推荐
[避坑] 可选
```

## 7.4 推荐程度

内部可以保存具体分数，但前端不展示 92/100 这种数字。

前端只展示：

```text
强推荐
推荐
可选
```

# 8. 预览片段定义

## 8.1 什么是预览片段

预览片段不是导出，也不是生成文件。

预览片段指：

> 用户点击候选片段后，左侧播放器跳到该候选的开始时间，并播放到结束时间附近，让用户人工确认这段是否值得导出。

例如候选片段是：

```text
41:05 - 42:50 · 1分45秒
```

用户点击「预览片段」后：

```text
左侧播放器跳到 41:05
播放到 42:50 附近
用户确认内容是否完整、开头结尾是否自然
```

## 8.2 预览的作用

预览用来确认：

```text
1. 这段内容是否真的完整
2. 开头是否突兀
3. 结尾是否自然
4. 画面和语气是否适合发布
5. 字幕是否需要修改
6. 是否值得导出
```

## 8.3 预览和导出的区别

| 操作      | 作用                   | 是否生成文件    |
| ------- | -------------------- | --------- |
| 预览片段    | 播放原视频对应时间段，人工确认      | 不生成       |
| 快速导出    | 本地生成 MP4 + SRT + TXT | 生成文件      |
| 生成带字幕视频 | 上传短片段到服务器烧录字幕        | 生成带字幕 MP4 |


```text

→ 上传短片到服务器
→ 服务器烧录字幕
→ 下载带字幕 MP4
```

## 18.3 回来继续处理

```text
打开私密项目链接
→ 系统显示候选片段仍在
→ 如果无法访问原视频
→ 提示重新选择同一个 MP4
→ 用户选择原视频
→ 继续预览和导出
```

# 19. MVP 前端最终定稿

MVP 前端采用：

```text
上传页 + 项目页
```

项目页采用：

```text
左右分栏
```

左侧：

```text
本地视频播放器
当前片段信息
文案 Tab
字幕 Tab
导出 Tab
```

右侧：

```text
TOP 5 候选片段
推荐优先 / 时间顺序切换
候选卡片默认态 / 展开态
查看更多候选
重新生成候选
```

核心定义：

```text
候选 = AI 推荐的可能值得二次发布的时间段
预览 = 用户人工确认这段是否值得导出
导出 = 真正生成 MP4 / SRT / TXT / 带字幕视频
```

核心边界：

```text
不做复杂剪辑软件
不做时间轴编辑
不做移动端
不上传原始完整视频
不展示后台技术日志
不让用户混淆预览和导出
```

# 19. 开发边界

必须遵守：

```text
原始完整视频不上传服务器
没有原视频时不能预览和切片
候选未生成时不能导出
预览不生成文件
快速导出在浏览器本地完成
生成带字幕视频只上传当前短片
不要把技术日志直接暴露给用户
```

不要做：

```text
复杂时间轴
专业剪辑器
移动端处理
素材库
自动发布
平台账号绑定
团队协作
复杂字幕样式编辑
```

# 20. 最终实现逻辑总结

核心路径：

```text
UploadPage
→ 选择本地 MP4
→ ProjectPage
→ 浏览器本地处理视频音频
→ 上传压缩音频
→ AI 分析 transcript
→ 生成候选片段
→ 用户点击预览片段
→ 左侧播放器播放对应时间段
→ 用户编辑文案/字幕
→ 用户快速导出或生成带字幕视频
```

核心状态关系：

```text
候选 = AI 推荐的时间段
预览 = 播放原视频对应时间段
导出 = 生成实际文件
```

核心组件关系：

```text
RightPanel 负责候选选择
LeftPanel 负责预览、编辑、导出
ProjectHeader 负责状态解释
ExportTab 负责文件生成
```

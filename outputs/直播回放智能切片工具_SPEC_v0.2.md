# 直播回放智能切片工具 SPEC v0.2

## 1. Problem Statement（问题陈述）

知识类主播在直播结束后，通常会得到一段 1-2 小时的完整直播回放视频。

但主播本人往往没有时间完整回看视频，也不清楚哪些片段适合二次分发。即使找到片段，还需要手动剪辑、加字幕、起标题、提炼摘要和金句，整个过程效率很低。

大量有价值的直播内容因此没有被复用成短视频素材。

核心问题是：

> 知识主播缺少一个低门槛工具，能够在不完整回看直播回放的情况下，快速发现、确认并导出可发布的知识切片。

---

## 2. Target User（目标用户）

MVP 第一版目标用户：

> 知识类主播本人。

典型用户包括：

* AI 教程主播
* 技术分享主播
* 商业/产品/运营类主播
* 教育/课程类直播创作者
* 个人知识 IP 创作者

MVP 不优先服务：

* 专业剪辑团队
* 多账号运营团队
* 带货直播团队
* 游戏/娱乐主播
* 企业内容团队

---

## 3. Proposed Solution（方案描述）

这是一个面向知识类主播的 **浏览器本地处理优先** 的直播回放智能切片工具。

用户选择本地 MP4 直播回放视频后，系统不会把原始视频上传到服务器。

浏览器端负责读取本地视频、抽取音频、压缩音频，并将压缩后的 MP3 音频上传到服务器。

服务器基于音频生成 transcript，并调用 LLM 分析 transcript，找出适合二次发布的知识片段。

系统默认生成 10 个候选片段，默认展示评分最高的 TOP 5。

每个候选片段包含：

* 起止时间
* 3 个候选标题
* 摘要
* 原文金句
* 推荐理由
* 风险提示
* 字幕文本
* 快速导出入口
* 带字幕成片导出入口

用户可以在项目页中预览候选片段、修改字幕文本、编辑标题/摘要/金句。

导出分为两种：

### 快速导出

浏览器使用用户本地原视频，按时间戳快速切片。

输出：

* MP4
* SRT 字幕文件
* TXT 文案文件

### 带字幕成片导出

浏览器先从本地原视频中切出 1-3 分钟短片，然后只上传这个短片到服务器。

服务器负责烧录字幕，并返回最终带内嵌字幕的 MP4。

原始 2 小时直播回放视频始终不上传服务器。

## 4. Core User Flow（核心用户流程）

### 4.1 选择本地视频

1. 用户进入上传页。
2. 用户选择本地 MP4 直播回放视频。
3. 页面提示：原始视频不会上传服务器。
4. 浏览器读取本地视频文件。
5. 系统创建一个私密项目页。

### 4.2 浏览器本地抽音频

1. 浏览器使用 FFmpeg.wasm 读取本地 MP4。
2. 从视频中抽取音频轨道。
3. 将音频压缩为 16kHz 单声道 MP3。
4. 浏览器上传压缩后的 MP3 到服务器。
5. 原始视频文件不上传。

### 4.3 服务器 AI 分析

服务器收到压缩音频后：

1. Python Worker 调用云端 ASR。
2. ASR 返回带时间戳 transcript。
3. 系统将 transcript 按固定时间分块。
4. 每个 chunk 保留 overlap。
5. LLM 分析每个 chunk，找出候选片段。
6. 系统合并、去重、排序候选片段。
7. 生成 TOP 10 候选。
8. 默认展示 TOP 5。

### 4.4 用户预览候选片段

1. 用户进入项目页。
2. 查看 TOP 5 候选片段卡片。
3. 点击某个候选片段。
4. 页面使用本地原视频播放器跳转到候选片段开始时间。
5. 播放到结束时间附近后停止或提示。
6. 用户判断该片段是否值得导出。

MVP 不默认生成低清预览文件。

### 4.5 用户编辑

用户可以编辑：

* 字幕文本
* 标题
* 摘要
* 原文金句

字幕编辑：

* 支持逐句修改字幕文本。
* 不支持调整字幕时间轴。

金句策略：

* 默认必须来自原始 transcript。
* 用户编辑金句只影响发布文案。
* 用户编辑金句不会改变视频音频，也不会改变字幕内容。

### 4.6 快速导出

用户点击快速导出后：

1. 浏览器使用本地原视频按时间戳切片。
2. 使用快速切片方式生成 MP4。
3. 根据字幕时间戳生成 SRT。
4. 生成 TXT 文案。
5. 用户本地下载文件。

单个快速导出输出：

* clip-01.mp4
* clip-01.srt
* clip-01.txt

批量快速导出 TOP 5：

* 浏览器本地切出 5 个 MP4。
* 生成 5 个 SRT。
* 生成 5 个 TXT。
* 浏览器本地打包 ZIP。
* 服务器不参与视频切片和 ZIP 打包。

### 4.7 带字幕成片导出

用户点击带字幕成片导出后：

1. 浏览器先从本地原视频中切出该短片。
2. 浏览器上传该 1-3 分钟短片到服务器。
3. Python Worker 使用 FFmpeg 烧录字幕。
4. 服务器生成带内嵌字幕 MP4。
5. 用户下载最终成片。

MVP 只支持单个切片的带字幕成片导出。

MVP 不支持 TOP 5 批量带字幕成片导出。

## 5. Clip Strategy（切片策略）

MVP 聚焦知识类直播。

切片标准参考 AI Highlight Clip 的高光评分思路，但针对知识直播做调整。

系统主要通过 transcript 文本判断片段价值。

优先寻找：

* 观点完整的片段
* 有明确结论的片段
* 有金句的片段
* 有案例的片段
* 有方法、步骤、避坑的片段
* 不依赖前后文也能独立观看的片段
* 适合二次发布的片段

MVP 暂不做：

* 音频情绪分析
* 语速分析
* 视频画面理解
* 多模态高光判断

---

## 6. Candidate Generation（候选生成）

系统对长 transcript 使用混合分块策略：

* 每 5 分钟左右切成一个 chunk。
* chunk 之间保留约 30 秒 overlap。
* 每个 chunk 内由 LLM 寻找候选片段。
* 最后统一合并、去重、排序。
* 系统内部生成 TOP 10。
* 默认展示 TOP 5。

每个项目允许免费重新生成候选 1 次。

后台记录：

* 是否重新生成
* 重新生成次数
* 是否点击候选片段
* 是否快速导出
* 是否带字幕成片导出
* 是否批量快速导出 TOP 5

## 8. Subtitle Strategy（字幕策略）

MVP 支持两种字幕交付方式：

### 快速导出

快速导出不烧录字幕。

输出：

* MP4
* SRT

用户可以将 SRT 导入剪映、B站、小红书、视频号或其他工具。

### 带字幕成片导出

用户选择带字幕成片时，服务器对短切片烧录字幕。

默认字幕样式：

* 底部居中
* 白字黑边或半透明背景
* 保证可读

MVP 支持：

* 逐句编辑字幕文本
* 修改错别字
* 修改专有名词
* 修改英文术语
* 修改人名、产品名

MVP 不支持：

* 字幕时间轴调整
* 多字幕模板
* 高级字幕动画
* 字幕样式设计器

## 9. Technical Constraints（技术约束）

### 9.1 输入限制

MVP 只支持：

* 本地 MP4 文件
* 单个视频最长 2 小时
* 单个文件最大 2GB
* 桌面端浏览器

MVP 不支持：

* 平台链接解析
* 自动下载直播回放
* 网盘导入
* 直播流实时处理

### 9.2 浏览器支持

MVP 推荐：

* 桌面端 Chrome
* 桌面端 Edge

其他桌面浏览器可尝试，但不保证完整支持。

移动端暂不支持。

### 9.3 浏览器本地处理

浏览器端负责：

* 读取本地 MP4
* 使用 FFmpeg.wasm 抽取音频
* 将音频压缩为 16kHz 单声道 MP3
* 本地播放器按时间段预览候选片段
* 根据时间戳快速切片
* 生成 SRT
* 生成 TXT 文案
* 批量快速导出时本地打 ZIP

浏览器端不负责：

* 复杂字幕烧录
* 视频重编码
* 复杂转场
* BGM
* 滤镜
* 封面设计

### 9.4 ASR

MVP 使用云端 ASR API。

架构上抽象为：

* ASRProvider

第一版支持 OpenAI-compatible ASR。

可先接：

* Groq Whisper

后续可替换：

* OpenAI Audio API
* 其他兼容 ASR 服务

### 9.5 LLM

MVP 使用云端 LLM。

架构上抽象为：

* LLMProvider

支持：

* OpenAI-compatible Provider
* 预留 Anthropic Provider

LLM 负责：

* 分析 transcript
* 发现候选切片
* 给候选片段评分
* 生成标题
* 生成摘要
* 提取原文金句
* 生成推荐理由
* 生成基础风险提示

### 9.6 技术架构

MVP 使用：

* Next.js
* Python Worker
* Neon Postgres
* 本地服务器文件存储
* 数据库 jobs 表轮询
* FFmpeg
* FFmpeg.wasm

### 9.7 Next.js 职责

Next.js 负责：

* 上传页
* 项目页
* 私密项目链接
* 候选片段展示
* 本地视频文件选择
* 本地预览控制
* 字幕编辑
* 标题/摘要/金句编辑
* 快速导出入口
* 带字幕成片导出入口
* 任务状态展示
* 轻量业务 API

### 9.8 Python Worker 职责

Python Worker 负责：

* 处理 ASR 任务
* 调用云端 ASR
* 处理 transcript
* 调用 LLM 分析切片
* 生成候选片段
* 生成标题/摘要/原文金句
* 接收用户上传的短切片
* 对短切片烧录字幕
* 生成最终带字幕 MP4
* 清理过期文件

Python Worker 不负责：

* 保存原始 2 小时视频
* 从原始视频抽音频
* 从原始视频切片
* 批量打包 TOP 5 视频

### 9.9 数据库

Neon Postgres 存储：

* project
* project token
* job status
* audio metadata
* ASR transcript
* transcript segments
* clip candidates
* subtitle lines
* edited title
* edited summary
* edited quote
* risk notice
* short clip upload path
* final subtitled output path
* expires_at

### 9.10 文件存储

MVP 内测阶段使用服务器本地磁盘存储。

服务器只保存：

* 压缩音频 MP3
* 用户上传的短切片
* 最终带字幕 MP4
* 临时字幕文件
* 临时处理文件

服务器不保存：

* 原始 2 小时直播回放视频

后续保留 StorageProvider 抽象，可迁移到：

* 阿里 OSS
* 腾讯 COS
* Cloudflare R2
* AWS S3

### 9.11 任务队列

MVP 先使用数据库 jobs 表。

Python Worker 定时轮询 jobs 表，领取待处理任务并更新状态。

后续如果并发、重试、调度复杂度上升，再迁移到：

* Redis + Celery
* Redis + RQ
* 其他任务队列

### 9.12 任务状态

MVP 显示阶段状态，不显示精确百分比。

阶段包括：

* 等待选择视频
* 本地抽取音频
* 本地压缩音频
* 上传压缩音频
* ASR 语音识别
* AI 分析切片
* 生成候选片段
* 等待用户确认
* 本地快速切片
* 上传短切片
* 烧录字幕
* 完成
* 失败

### 9.13 失败处理

MVP 显示失败阶段和简单原因，并允许重试失败阶段。

例如：

* 本地抽音频失败：提示浏览器或视频格式可能不支持
* 音频上传失败：允许重新上传压缩音频
* ASR 失败：允许重试 ASR
* LLM 分析失败：允许重试切片分析
* 本地切片失败：提示用户重新选择原始视频或换 Chrome/Edge
* 字幕烧录失败：允许重新上传短切片或重试烧录

## 10. Storage & Privacy（存储与隐私）

MVP 不做永久素材库。

保存策略：

* 原始 2 小时视频：不上传服务器，不保存
* 压缩 MP3 音频：最多保存 24 小时
* ASR 成功后，音频可以提前删除
* 短切片原文件：保存 24 小时
* 最终带字幕 MP4：保存 7 天
* transcript：保存 7 天
* 候选片段数据：保存 7 天
* 标题/摘要/原文金句/字幕文本：保存 7 天

项目访问方式：

* MVP 不强制登录
* 上传后生成私密项目链接
* 当前浏览器保存项目记录
* 用户可复制项目链接回来查看

安全边界：

* 项目链接必须包含高随机 token
* 知道链接的人可以访问项目页
* MVP 不做账号权限系统
* MVP 不做团队协作权限管理

由于原始视频不上传服务器，用户回来后如果浏览器无法继续访问原视频，需要重新选择同一个原始 MP4 文件。

## 11. Non-goals（明确不做的事）

MVP 暂不做：

1. 不做直播实时切片
   只处理直播结束后的本地回放视频。

2. 不做平台链接解析 / 自动下载回放
   不支持 B站、抖音、视频号、YouTube 链接解析。

3. 不做自动发布
   不接抖音、视频号、B站账号，不做自动投稿。

4. 不做竖屏智能裁剪
   保留原视频比例，横屏进横屏出，竖屏进竖屏出。

5. 不做账号系统 / 团队协作
   MVP 用私密项目链接，不做登录、多人权限、素材库。

6. 不做永久存储
   原视频不上传；压缩音频、短切片和结果文件都按期限自动删除。

7. 不做专业剪辑能力
   不做时间轴、多轨、转场、BGM、复杂字幕动画。

8. 不做复杂平台文案包
   只生成 3 个标题、摘要、原文金句；后续再做平台模板。

9. 不做浏览器端复杂重编码
   浏览器端主要做抽音频、快速切片、生成 SRT，不承担复杂字幕烧录。

10. 不做移动端视频处理
    MVP 推荐桌面端 Chrome / Edge。

## 12. Success Criteria（成功标准）

MVP 第一阶段成功标准：

使用 10 个知识直播回放视频进行内测。

成功条件：

* 至少 7 个视频能产出至少 1 个用户愿意导出的有效切片。
* 用户不需要完整回看原视频。
* 用户能在 30 分钟内完成候选筛选、字幕修改和至少 1 个快速切片导出。
* 用户可以成功导出 MP4 + SRT + TXT。
* 有一部分用户愿意上传第二个视频。

核心指标：

> 10 个测试视频中，至少 7 个能导出 1 个有效切片。

辅助指标：

* 本地音频抽取成功率
* 音频上传成功率
* ASR 成功率
* 候选生成成功率
* 用户是否点击候选片段
* 用户是否编辑字幕
* 用户是否快速导出单个切片
* 用户是否批量快速导出 TOP 5
* 用户是否使用带字幕成片导出
* 用户是否点击重新生成候选
* 用户是否上传第二个视频

---

## 13. MVP Summary（一句话总结）

这是一个面向知识类主播的浏览器本地优先直播回放智能切片工具。

用户选择本地 MP4 直播回放视频后，原始视频不上传服务器。浏览器本地抽取并压缩音频，服务器基于音频生成 transcript 并用 LLM 推荐高价值知识切片。用户在项目页预览候选片段后，可在浏览器本地快速导出 MP4 + SRT + 文案；如需带内嵌字幕成片，则只上传该短切片到服务器完成字幕烧录。

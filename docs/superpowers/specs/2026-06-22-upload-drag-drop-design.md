# Clipwise 上传页拖拽文件设计

## 目标

在现有上传页增加 MP4 拖拽入口，让没有使用系统文件选择器习惯的用户可以直接把视频拖到页面中，同时保留原有点击选择方式。

## 交互设计

上传卡片中增加一个独立拖拽区，整块区域可点击：

- 默认文案：`拖拽 MP4 到这里`
- 辅助文案：`或点击选择本地文件`
- 拖入文件时，边框和背景使用蓝色高亮，并显示 `松开即可选择`
- 放下有效文件后，复用现有文件信息、重新选择和开始分析流程
- 点击拖拽区时，打开系统文件选择器
- 已选择文件后仍允许把另一个文件拖入，新的有效文件替换旧文件

不把整个页面设置为拖拽区域，避免用户经过页面时误选文件。

## 文件校验

拖拽和点击选择共用现有 `validateVideoFile`：

- 仅接受 MP4
- 最大 2GB
- 校验失败时不保留无效文件
- 页面显示现有错误提示

浏览器一次拖入多个文件时，只处理第一个文件。

## 可访问性

- 拖拽区使用按钮语义，可通过键盘聚焦和触发
- 原生文件输入继续保留明确标签
- 拖入高亮不作为唯一反馈，同时更新可见文字
- 错误信息继续使用 `role="alert"`

## 状态和组件边界

继续由 `UploadPageClient` 管理：

- `file`
- 上传状态
- 校验错误
- 是否正在拖入文件

暂不拆分新的上传组件，避免为单一交互增加不必要的组件层级。

## 测试

组件测试覆盖：

- 拖入有效 MP4 后显示文件名和开始分析按钮
- 拖入无效文件后显示错误
- 拖入时出现 `松开即可选择`
- 点击拖拽区仍能触发现有文件输入

端到端测试继续验证选择文件后进入项目页；拖拽行为以组件测试为主。

## 非目标

- 文件夹拖拽
- 多文件上传
- 上传进度
- 全页面拖拽
- 移动端视频处理
